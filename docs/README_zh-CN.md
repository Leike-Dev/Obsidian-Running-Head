用优雅的博客风格元数据页眉来改变你的笔记！📝✨

**Running Head** 是一款 Obsidian 插件，它可以自动为你的笔记添加可自定义的页眉，显示发布日期、预计阅读时间、更新标签、笔记导航路径（面包屑）、标签页导航、滚动进度条以及任何你想要的自定义 frontmatter 属性。

## 功能

- **📝 多种布局**：在 **Blog 样式**（路径在标题上方，元数据在下方）和 **Wiki 样式**（元数据在标题上方，路径在下方）之间选择。
- **⏱️ 阅读时间与日期**：根据您的语言环境自动计算并显示阅读时间和格式化日期（支持亚洲语言）。
- **🧩 自定义字段**：在页眉中显示任何 YAML 属性，如漂亮的“胶囊”、链接或文本，以及复选框 —— 并可单独将其放置在标题**上方**或**下方**。
- **📁 文件夹范围**：通过在特定文件夹中隐藏指定的自定义字段来保持仓库的整洁（支持每个字段多个文件夹）。
- **🍞 笔记路径**：通过可点击的文件夹路径轻松导航，准确显示笔记所在位置，并可选地高亮显示当前文件夹。
- **📊 滚动进度条**：固定在笔记顶部的一条优雅进度条，实时跟踪你的阅读进度。
- **🎨 自定义颜色**：简单且独立地定义更新标签、笔记路径或进度条的颜色。
- **📅 灵活的日期格式**：从18种可用语言环境中选择格式化语言，或者使用 [Moment.js](https://momentjs.com/) 语法定义自定义格式。
- **💾 数据管理**：将完整设置导出为 JSON，并轻松导入到另一个仓库或设备。
- **🤝 强大的集成**：与 **[Typify](https://github.com/Leike-Dev/Obsidian-Typify)** 插件完美集成（继承你自己制作的漂亮胶囊样式）。
- **🌍 国际化**：界面已完全翻译为英语、葡萄牙语（巴西）、西班牙语、法语和简体中文（只需在其中一种语言下使用 Obsidian）。

## 如何使用

### 1. 配置 frontmatter 属性

在插件设置中，定义用于以下内容的 YAML 键：

- **创建日期**（默认：`date`）
- **更新日期**（默认：`updated`）

*或你首选的其他日期类型属性。*

### 2. 自定义布局与外观

- **标题大小**：调整你笔记的标题大小。
- **元数据大小**：调整配置在页眉中显示的元数据的大小。
- **页眉布局**：在 Wiki（元数据在标题上方，路径在下方）和 Blog（路径在标题上方，元数据在下方）之间选择。
- **笔记路径**：在标题上方显示笔记的文件夹路径（位于仓库根目录的笔记不显示）。
- **高亮当前文件夹**：将强调色应用于路径的最后一个片段。
- **滚动进度条**：在笔记顶部显示阅读进度条。
- **显示更新标签**：当笔记在创建后被修改时显示最后修改日期。
- **文件夹高亮颜色**：自定义路径高亮颜色（留空 = 主题强调色）。
- **进度条颜色**：进度条颜色（留空 = 主题强调色）。
- **更新标签颜色**：标签背景颜色（留空 = 主题默认颜色）。

### 3. 配置日期与阅读

- **格式化语言**：在18种可用语言中进行选择（en-US, ja-JP, pt-BR, es-ES 等）。
- **自定义格式**：使用 Moment.js 语法（例如 `DD/MM/YYYY`, `MMMM D, YYYY`）。留空则使用区域默认值。
- **缩写月份名称**：选择缩写月份名称（例如："1月"而非"一月"）。
- **显示阅读时间**：启用或禁用显示在日期旁边的预计时间。
- **阅读速度**：用于计算估计时间的每分钟字数（默认：`200`）（支持亚洲语言）。

### 4. 添加自定义字段

1. 转到 **设置 > Running Head**。
2. 在“自定义字段”下，点击 **添加**。
3. 配置字段选项：
   - **YAML 键**：要显示的 frontmatter 属性（例如 `Author`, `Category`, `tags`）。
   - **显示标签**：在值之前显示的可选文本。
   - **显示标签**：启用后在笔记中显示标签。
   - **位置**：选择 **标题上方** 或 **标题下方**。
   - **在文件夹中隐藏**：在指定文件夹中隐藏该字段（支持多个文件夹）。
4. 使用 **管理** 按钮来编辑、重新排序或删除现有字段。

### 5. 配置标签导航

1. 转到 **设置 > Running Head**。
2. 在 **标签导航** 下，配置标签的视觉样式（**下划线**、**药丸** 或 **极简**）。
3. 点击 **新建标签属性** 下的 **添加**，注册一个 frontmatter 属性键（例如 `tabs-home`）。
4. 在你的笔记 frontmatter 中，将此属性定义为 **列表**（List）类型。
5. 在列表中，你可以添加：
   - 目标笔记链接（双链/Wiki格式）：`"[[目标笔记]]"` 或 `"[[目标笔记|别名]]"`
   - 自定义标签名称（可选）：`"[name, 我的自定义名称]"`
   - 任意 Lucide 图标（可选）：`"[icon, home]"`（提示：点击设置中的图标选择器可以复制正确的图标标签到剪贴板）

> [!Note]
> - 列表内的标签（`[name, ...]`、`[icon, ...]` 和 `[[Link]]`）的顺序并不影响功能。
> - 如果列表中存在多个笔记链接，标签将指向最后添加的链接。

### 6. 数据管理

- **导出**：生成包含所有当前配置的 JSON 以便复制或保存。
- **导入**：粘贴配置 JSON 以快速应用到另一个仓库或设备。

就这样，你的元数据现在优雅地显示在笔记中了。 ✨

## 安装

### 手动安装
1. 下载最新版本：`main.js`、`manifest.json` 和 `styles.css`。
2. 在 `.obsidian/plugins/` 目录中创建一个名为 `running-head` 的文件夹。
3. 将文件粘贴到那里。
4. 重新加载 Obsidian 并在 **设置 > 第三方插件** 中启用该插件。

## 提示

> [!Tip]  
> 如果你安装了 **[Typify](https://github.com/Leike-Dev/Obsidian-Typify)** 插件，Running Head 会自动检测并应用你的 Typify 样式到元数据页眉中的匹配胶囊上（如果你想要的话）。

> [!Tip]  
> Kepano 制作的 Minimal 主题，为了让它尽可能达到最佳状态，也是让我最头痛的一个主题。现在它在这个主题上完美运行了，好吗？

> [!Tip]  
> 使用 **导出/导入** 功能在仓库之间共享你的设置，或者在尝试新调整之前备份你的配置。

## 开发

如果你想自己编译该插件，请按以下步骤操作：

1. 克隆此仓库。
2. 运行 `npm install`。
3. 运行 `npm run dev` 启动监听模式编译。
